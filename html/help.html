Ice Detector Information and Help

The purpose of this device is to detect the build up of ice and report it to the users before it becomes significant enough to cause disruptions.

Detailed guide on setting up remote access into the Raspberry Pi

The ice detector program is run on a Raspberry Pi that uses a connected infrared camera to sense and record the scene. Calculations are performed on the image to measure the intial conditions which are set as base conditions, then any subsequent calculations will compare the scene at the time to these conditions.

Once the ice buildup has met a predetermined threshold, the detector will be triggered and send an email notify the staff that there is a significant amount of ice accretion. The detector can then be reset or turned off by the users once the ice has been removed, which will then be ready for the next event that triggers it.

To communicate with the software, a Gmail account has been assigned and configured to each specific device that allows the user to send and receive information pertaining to that specific detector. Information can be received by sending message to the detector, the software will then read the contents of that message and send the appropriate response.

How to send a message

To prevent misuse, only the account assigned to the ice detector can query itself for information. This prevents unwanted uses from unnecessarily reset/altering the devices settings. The subject line must be "choice" for the software to tead the query. Then for the text line, a simple one worded message that describes the main behaviour of the query is typed and the message will then be ready to send.

Below is a table of the options currently available to send as text.

NOTE: The text field is case-sensitive. Type the message as it is shown below.

Message Description
reset Reset the base conditions to be compared against and return the state of the ice detector to continue detecting for ice build-up
toggle Depending on the current state of the detector, the state will be toggled on/off. Used for turning off the program when it is no longer necessary. e.g. warmer months, off-season
report Compiles data such as weather forecast, current scene of device etc.
help Sends this document to the user to explain functions and purpose of device.
